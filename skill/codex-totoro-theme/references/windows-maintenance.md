# Windows 维护与踩坑记录

仅在排查主题、适配新 Codex 构建或修改界面时读取本文件。正常安装直接运行 `scripts/windows-theme.ps1`。

## 安全边界

- Windows 版使用 Chromium DevTools Protocol 运行时注入 CSS 和本地图片，不写入 Microsoft Store 的安装目录，也不修改 `app.asar`。
- 调试端口必须绑定 `127.0.0.1`，并使用随机空闲端口。
- 只有 Codex 首次启动时传入远程调试参数才会生效。检测到 Codex 已运行时必须提示用户正常退出，禁止自动结束 Codex。
- `packageVersion` 与完整 `app.asar` SHA-256 都匹配时才能启动主题。Codex 升级后先重新验证，禁止只删除哈希门禁来“兼容”。
- 官方快捷方式始终启动未注入的官方界面；`Codex - Totoro Night` 快捷方式才启动主题。

## 实际遇到的问题

### 顶栏与窗口顶部之间出现空隙

**现象：** 任务标题栏整体向下约 30-40px，没有贴住应用顶部。

**原因：** 把内容区的原生安全间距同时应用到了任务标题栏，或给主题标题容器重复设置了 `top`、`margin-top`、`padding-top`。

**修复：** 只对任务内容保留安全间距；标题栏本身使用原生定位并归零额外上边距。检查标题栏顶部与父容器顶部差值应接近 `0`。

### 宠物后面出现半透明矩形

**现象：** 宠物周围有灰黑色透明底板，看起来像图片没有抠干净。

**原因：** 底板通常来自宠物宿主、伪元素、阴影或毛玻璃，而不是透明 WebP 本身。只处理图片透明通道无法解决。

**修复：** 对宠物宿主、内部画布和相关 `::before`/`::after` 同时清除 `background`、`box-shadow`、`backdrop-filter`、`border`。保留宠物交互容器，不要把点击区域一起隐藏。用像素检查确认精灵四角 alpha 为 0。

### 文字覆盖右上角按钮

**现象：** `TOTORO NIGHT` 或其他品牌文字压住窗口控制、输出面板或工具按钮。

**原因：** 在宽泛容器上使用全局伪元素和绝对定位，容器尺寸变化后文字跨入原生按钮区域。

**修复：** 删除全局伪元素品牌字。需要标识时只挂载到明确、独占的主题区域，并给按钮区保留可测量的空间。回归检查窄窗口和右侧面板展开状态。

### 菜单已挂载但切换器不出现

**现象：** 点击头像后看不到 Free/Plus/Pro 按钮，DOM 观察器也没有再次触发。

**原因：** Codex 会预先把个人菜单以不可见状态挂载，打开菜单只改变可见性，没有新增节点；只监听 `childList` 会漏掉。

**修复：** 保留 MutationObserver，同时在已有的低频同步循环中调用 `addPlanPicker()`。菜单可见后最多 500ms 补入按钮，避免新建高频定时器。

### 套餐识别与手动切换互相覆盖

**现象：** 用户点击 Plus 或 Pro 后，稍后又被自动识别改回其他颜色。

**原因：** 自动识别每次同步都覆盖了用户选择。

**修复：** 使用两个 localStorage 键：

- `codex-totoro-night-plan`：保存 `free`、`plus` 或 `pro`。
- `codex-totoro-night-plan-manual`：用户点过切换器后保存 `true`。

存在手动标记时不要再自动覆盖。未知账号默认 Free；可从用量菜单的“升级至 Plus/Pro”文案推断当前层级，但只能在没有手动选择时使用。

### 宠物仍显示旧角色

**现象：** 主题背景已更新，但宠物还是旧精灵或其他角色。

**原因：** Codex 的宠物画布可能晚于主题脚本重绘，或者只替换了背景，没有持续同步原生宠物宿主。

**修复：** 注入器需要在页面重载和宠物节点重建后重新应用龙猫 spritesheet。不要修改官方宠物资源槽；运行时覆盖便于恢复，也不会留下 ASAR 备份问题。

### 宠物列表的所有缩略图都变成龙猫

**现象：** 宠物设置页中的 Codex、Dewey、Fireball 等不同宠物全部显示同一个龙猫缩略图。

**原因：** 旧逻辑在主窗口全量扫描 `.codex-avatar-root` 和所有包含 `codex-spritesheet` 的背景，误把宠物目录预览也当成当前活动宠物。原始内联背景为空时又用真假值判断备份状态，下一轮可能把龙猫地址反写成“原始背景”。

**修复：** 只允许 `initialRoute=/avatar-overlay` 的独立宠物浮窗执行原生 spritesheet 替换；主窗口只保留主题自带的 composer 小宠物。备份判断必须使用 `hasAttribute`，不能判断字符串真假。每次注入前先恢复历史内联值；恢复时只把与当前主题 pet URI 完全相同的值视为污染，不能笼统删除其他 data URI，因为用户自定义宠物本身也可能是 data URI。

### 选择器在更新后失效

**现象：** 某些区域突然恢复官方样式，或 CSS 误伤无关按钮。

**原因：** 使用了压缩构建生成的类名、按节点顺序定位，或过宽的文本/标签选择器。

**修复：** 优先使用 Codex 暴露的稳定 `data-*` 属性，例如项目行、任务行和任务标题标记。文本只用于补充识别动态菜单，不能作为全局样式入口。

### 切到原版后无法打开主题面板

**现象：** 禁用龙猫 CSS 后，顶部主题入口也一起消失。

**原因：** 把顶部选择面板样式写进了会被禁用的主题 stylesheet。

**修复：** 顶部主题面板必须使用独立、始终启用的 `codex-totoro-theme-switch-style`。切到原版只设置主题 stylesheet 的 `disabled`，同时清理仪表盘、配色菜单、附加宠物和原生宠物覆盖；主题面板本身不能被清理。

### 宠物浮窗出现重复主题按钮

**现象：** 主窗口之外的宠物浮窗或快捷窗口顶部也出现切换器。

**原因：** 注入器会连接所有可检查的 page 目标，未区分主窗口和带 `initialRoute` 的辅助窗口。

**修复：** 只在没有 `initialRoute` 查询参数的主窗口创建主题面板。主题选择写入 `codex-totoro-theme-mode`，其他窗口通过 `storage` 事件同步启用或禁用主题。新增主题时扩展 `themeCatalog` 和对应的主题激活逻辑，顶部面板结构无需重做。

### 自定义主题背景

顶部主题面板最下方提供 **自定义图片**。只接受 PNG、JPEG 和 WebP，文件上限为 `15 MB`；推荐原图为 `1536 × 1024` 横图，主体放在中间偏右且不要嵌入重要文字。浏览器使用 Canvas 居中 `cover` 裁切到 `1240 × 889`，再按 `0.86` 质量编码为 WebP。

生成的数据 URI 只保存在 Codex 当前来源的 localStorage 键 `codex-totoro-custom-hero`，不发送到网络；通过 CSS 变量 `--totoro-night-hero-image` 同时替换空任务首页横幅和 `.main-surface` / `.browser-main-surface` 工作页画布。工作页必须继续叠加深色渐变以保证对话和代码可读，不要让用户图片覆盖侧边栏或宠物。上传图片会自动启用龙猫主题，**恢复默认** 删除该存储键并立即回退到内置背景。

上传逻辑需要同时防止超大文件和超大解码画布：除 `15 MB` 文件限制外，解码后不得超过 `40,000,000` 像素。存储配额不足、损坏图片和不支持的格式必须在面板内给出状态提示，不能让异常中断主题同步循环。自定义图片变化通过 `storage` 事件同步；切到官方主题或执行运行时清理时移除页面上的 CSS 变量，但不要修改 Microsoft Store 包。

## 新构建适配流程

1. 正常升级 Codex，运行 `windows-theme.ps1 check` 记录包版本和 ASAR 哈希。
2. 不开启主题观察官方界面，确认标题栏、侧栏、空任务快捷操作、输入区、输出面板、个人菜单和宠物宿主结构。
3. 在临时副本中更新选择器，禁止先改当前用户正在使用的已安装 Skill。
4. 用新构建分别验证空任务、已有任务、右侧输出面板、窄窗口、个人菜单和宠物动画。
5. 测量标题栏顶部间距；检查主题入口位于 36px 菜单栏内，展开面板无文字溢出并支持点击外部和 `Esc` 关闭；分别上传 PNG/JPEG/WebP、重启验证自定义背景持久化，再恢复默认；检查宠物四周无底板且宠物目录缩略图保持各自原图；依次切换原版/龙猫和 Free/Plus/Pro。
6. 确认 `restore` 能移除实时样式和主题快捷方式，官方快捷方式未受影响。
7. 最后再更新 `windows-theme.ps1` 中的版本与 SHA-256，并重新生成 `.skill` 包。

```powershell
node --check payload/src/windows-injector.mjs
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/windows-theme.ps1 check
python C:\Users\<user>\.codex\skills\.system\skill-creator\scripts\quick_validate.py .
```

从仓库根目录运行 `npm test` 和 `npm run package:totoro`。压缩包内第一层必须是 `codex-totoro-theme/`，不能把仓库绝对路径、临时截图、运行时端口或 `runtime.json` 打进发布包。
